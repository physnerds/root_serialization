4 /*---------------------------------------
5 The ProxyVector class is intended to look like a contiguous container
6 holding base classes by value. The elements of a given ProxyVector are
7 all the same actual type (which inherit from the given base class).
8 
9 The actual underlying implementation uses the Pimpl idiom via the use of
10 ProxyVectorImpBase and ProxyVectorImp. The elements are stored in the
11 ProxyVectorImp via std::vector<> holding the concrete class by value.
12 
13 Using ProxyVector instead of a std::vector<std::unique_ptr<Base>> allows
14 for better memory cache locality as each successive element of the container
15 is guaranteed to be next to each other in memory. Also requiring all the
16 elements to be of the same type means virtual function lookups should be
17 cached on the first element request and be reused for each subsequent call
18 when looping over all elements.
19  ---------------------------------------*/
